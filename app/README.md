# app/

Flutter application for iOS and Android. UI-only layer — all business logic lives in `../core/`
(Rust) and is exposed to Dart through the bridge abstraction.

## Structure

```
app/
├── lib/
│   ├── bridge/        Abstract Bridge interface + BridgeStub (in-memory mock for dev)
│   ├── state/         Riverpod providers — own all bridge calls, no direct bridge use in UI
│   ├── presentation/  Widgets and screens — no bridge calls, reads from state/ only
│   ├── platform/      Push registration, local notifications, share sheet, deep links
│   └── plugin_api/    Plugin extension points (stubbed, Phase 1)
├── rust_bridge/       flutter_rust_bridge glue — see rust_bridge/README.md
├── ios/               Xcode project (generated by Flutter tooling, hand-editable)
└── android/           Gradle project (generated by Flutter tooling, hand-editable)
```

## Running

```bash
flutter pub get
flutter run                   # picks up a connected device
flutter run -d <device-id>    # target a specific simulator/device
flutter devices               # list available targets
```

## Bridge

The app currently runs on `BridgeStub` (in-memory mock data) so the UI can be developed and
tested without the Rust bridge compiled in. To activate the real bridge once
`flutter_rust_bridge_codegen` is available:

1. Uncomment `flutter_rust_bridge: ^2.0.0` in `pubspec.yaml`
2. Run `flutter_rust_bridge_codegen generate` in this directory
3. Implement `BridgeFrb implements Bridge` in `lib/bridge/`
4. Update `bridgeProvider` in `lib/state/bridge_provider.dart` to return `BridgeFrb()`
